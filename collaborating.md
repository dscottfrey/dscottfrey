# GitHub: Sharing Private Repositories

Use these steps to grant a specific person access to your private code without making it public to the world.

### 1. Access Repository Settings
* Log in to [GitHub.com](https://github.com) and navigate to your private repository.
* Click the **Settings** tab (the gear icon in the top navigation bar).

### 2. Add a Collaborator
1. In the left-hand sidebar, under the **Access** section, click **Collaborators**.
2. Click the green **Add people** button.
3. In the search box, type the person's **GitHub username** or **email address**.
4. Select the correct user from the dropdown and click **Add [username] to this repository**.

### 3. Verification
* **Invitation Sent:** GitHub will send an invitation to the user. 
* **Action Required:** The recipient **must accept** the invitation via the email GitHub sends them or by visiting the repository URL. They will not see the code until they click **Accept**.
* **Permissions:** By default, they are given **Write** access. If you only want them to view the code, you can change the "Role" dropdown to **Read** next to their name in the Collaborators list.

---

### 💡 Important Notes
* **Free Tier:** You can have unlimited collaborators on private repositories with a standard GitHub Free account.
* **Security:** If you ever need to revoke access, simply return to the **Collaborators** page and click the **Remove** (trash can) icon next to their name.
* **Visibility:** To everyone else on the internet, the repository remains completely hidden and returns a "404 Not Found" error if they try to visit the URL.
